# Plan 02: Authentication API — OAuth2 + Email + JWT + Profile

## Objective

Build the complete auth API that the React frontend will consume. Covers social login (Google, Facebook, Instagram via drf-social-oauth2), email/password registration, JWT management with httpOnly refresh cookies, and profile endpoints including the "is profile complete?" check needed for the checkout gate.

## Context

- Plan 01 must be complete — User model, groups, and Django scaffold exist.
- drf-social-oauth2 bridges python-social-auth and django-oauth-toolkit. It provides `/auth/convert-token/` out of the box.
- For auto-merge: add `social_core.pipeline.social_auth.associate_by_email` to SOCIAL_AUTH_PIPELINE.
- Refresh tokens must be in httpOnly cookies (CONTEXT.md decision). Access tokens returned in response body.
- Profile is "complete" when: phone is not blank AND at least one Address exists (CONTEXT.md: banner until checkout).
- SameSite must be 'Lax' not 'Strict' for OAuth redirect callbacks to work (RESEARCH.md).
