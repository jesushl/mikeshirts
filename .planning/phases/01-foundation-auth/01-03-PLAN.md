# Plan 03: React Frontend Scaffold — Vite + Tailwind + Routing + Auth Infrastructure

## Objective

Create the complete React frontend scaffold with Vite, Tailwind CSS dark theme, three-level routing with lazy loading, Axios API client with JWT interceptor, and Zustand auth store. This gives Plan 04 (auth UI) all the infrastructure it needs.

## Context

- Plan 01 must be complete (backend running on :8000).
- Plan 02 can run in parallel — frontend scaffold doesn't need auth API to exist yet, just knows the endpoint shapes.
- CONTEXT.md decisions: dark theme, Inter/Plus Jakarta Sans, header fijo estándar, 3-level routing with lazy loading.
- Axios interceptor reads access token from Zustand store, on 401 calls /api/auth/token/refresh/ (reads httpOnly cookie), retries original request.

<task id="01-03-03">
<name>Create Axios API client with JWT interceptor and Zustand auth store</name>
<files>
- frontend/src/services/api.ts
- frontend/src/store/authStore.ts
- frontend/src/hooks/useAuth.ts
</files>
<action>
1. In api.ts:
   - Create Axios instance with baseURL '/api/' (proxied by Vite in dev).
   - Request interceptor: attach `Authorization: Bearer <token>` from authStore if token exists.
   - Response interceptor: on 401, call /api/auth/token/refresh/ (cookie sent automatically), store new access token, retry original request. If refresh fails → clear auth state, redirect to /login.
   - Handle race condition: if multiple requests get 401 simultaneously, only one refresh call should happen (use a promise queue).
2. In authStore.ts (Zustand):
   - State: user (User | null), accessToken (string | null), isAuthenticated (boolean), isProfileComplete (boolean)
   - Actions: setAuth(user, token), clearAuth(), setProfileComplete(bool), checkAuth() (calls refresh endpoint on app load)
   - Persist: nothing (access token in memory only; refresh is httpOnly cookie)
3. In useAuth.ts:
   - Custom hook wrapping authStore with convenience methods: login(), register(), logout(), socialLogin(provider)
   - login/register call the API, then setAuth with response
   - socialLogin redirects to the OAuth provider URL
   - logout calls /api/auth/logout/, clears store
</action>
<verify>
- `authStore.getState().isAuthenticated` starts as false
- After calling checkAuth() with a valid refresh cookie, isAuthenticated becomes true
- After 401, interceptor refreshes and retries successfully
- After logout, accessToken is null and isAuthenticated is false
</verify>
