# Plan 04: Auth UI — Login, Register, Profile Completion, Account Page

## Objective

Build all auth-related frontend pages connecting to the API from Plan 02. Complete the end-to-end flow: user sees login page → registers or social logins → sees profile completion banner → fills phone + address → banner disappears → can access account page. This closes Phase 1.

## Context

- Plan 02 (auth API) and Plan 03 (React scaffold) must both be complete.
- CONTEXT.md: redirect flow for OAuth (not popup), banner persistent until checkout (not blocking navigation), registration available in header at all times.
- Social login flow: click button → redirect to provider → return to callback URL → frontend extracts code → calls /api/auth/convert-token/ → receives JWT.
- Profile is "complete" when phone is set AND at least one address exists.

<name>Create profile completion flow and persistent banner</name>
<files>
- frontend/src/pages/CompleteProfilePage.tsx

</files>
<action>
1. ProfileBanner component:
   - Shown in PublicLayout when authStore.isAuthenticated && !authStore.isProfileComplete
   - Sticky below header, full width, accent color background
   - Text: "Completa tu perfil para poder hacer compras" + button "Completar perfil" → navigates to /cuenta/completar-perfil
   - Dismiss button (hides for current session only, reappears on page reload)
2. CompleteProfilePage:
   - Form with: phone (required), RFC (optional), address fields (street, ext_number, int_number optional, neighborhood, city, state, zip_code)
   - On submit: PATCH /api/auth/me/ for phone+rfc, POST /api/auth/me/addresses/ for address
   - On success: update authStore.isProfileComplete = true, redirect to /cuenta
   - Validation: phone format, required address fields
3. Update PublicLayout to include ProfileBanner.
4. On app load (main.tsx or App.tsx): call authStore.checkAuth(). If authenticated, also call /api/auth/me/profile-complete/ and set isProfileComplete.
</action>

- Clicking "Completar perfil" navigates to form
- Submitting form with valid data makes banner disappear
- Refreshing page after completing profile: no banner
- Dismissing banner hides it until page reload
