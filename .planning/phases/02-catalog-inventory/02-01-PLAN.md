# Plan 01: Catalog & Inventory Models + Django Admin

## Objective

Create the `catalog` and `inventory` Django apps with all the data models needed to represent products, variants, images, categories, and stock. Configure Django Admin so the team can start managing the catalog immediately.

## Context

- Phase 1 complete â€” User model, auth API, groups exist.
- Products have variants (size Ă— color). Each variant is a purchasable SKU.
- Images belong to Product (not Variant). One image is flagged as main.
- Size chart is a JSONField on Product (measurements per size).
- Stock is 1:1 with Variant. production_days enables the "made to order" display.
- Color and size are CharField with predefined choices (not separate models).
- Slug is auto-generated from name, editable in admin.
- ADM-05/ADM-06 are handled via Django Admin for now (custom panel is Phase 4).
