# Plan 02: Public Catalog API — List, Detail, Filters, Search

## Objective

Build the REST API that the React frontend will consume to display the catalog. Covers product listing with pagination, filtering (category, size, color, price range), text search, product detail by slug with full variant/stock info, and category listing.

## Context

- Plan 02-01 must be complete — all catalog and inventory models exist.
- All catalog endpoints are public (no auth required) — read-only.
- Products are filtered by `is_active=True` in all public queries.
- Stock availability is computed per variant and included in the product detail response.
- django-filter is the standard way to do query param filtering in DRF. Needs to be added to requirements.
- Pagination: use DRF's PageNumberPagination with page_size=12 (standard grid).

<task id="02-02-03">
<name>Create filters and views for catalog API</name>
<files>
- backend/apps/catalog/filters.py
- backend/apps/catalog/views.py
</files>
<action>
1. In filters.py — `ProductFilter(django_filters.FilterSet)`:
   - category: filter by category slug (CharFilter, field_name='category__slug')
   - size: filter products that have a variant with this size (CharFilter, method)
   - color: filter products that have a variant with this color (CharFilter, method)
   - min_price / max_price: filter by base_price range (NumberFilter)
2. In views.py:
   - `CategoryListView(ListAPIView)`: returns all categories, no auth required.
   - `ProductListView(ListAPIView)`: active products, ProductListSerializer, ProductFilter, search on name+description, ordering by created_at/price.
   - `ProductDetailView(RetrieveAPIView)`: lookup by slug, ProductDetailSerializer, no auth required.
3. All views use `permission_classes = [AllowAny]`.
</action>
