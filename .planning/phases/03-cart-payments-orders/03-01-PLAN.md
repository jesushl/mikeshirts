# Plan 01: Cart — Session + DB + Merge + API

## Objective

Build the complete cart system. Anonymous users get a session-based cart; authenticated users get a DB cart. On login, the frontend calls merge to combine both. All operations validate stock availability.

## Tasks

<task id="03-01-01">
<name>Create cart app with models</name>
<action>
1. Create cart app package.
2. Cart model: user (OneToOne, unique), created_at, updated_at.
3. CartItem model: cart FK, variant FK, quantity (PositiveIntegerField, min 1). Unique together: (cart, variant).
4. Add 'apps.cart' to INSTALLED_APPS, makemigrations, migrate.
</action>
<done>[ ]</done>
</task>

<task id="03-01-02">
<name>Build session cart service</name>
<action>
1. services.py — SessionCart class that wraps request.session:
   - CART_SESSION_KEY = 'cart'
   - add(variant_id, quantity) — validates stock, adds/updates
   - remove(variant_id)
   - update_quantity(variant_id, quantity)
   - clear()
   - get_items() — returns list of {variant_id, quantity} dicts
   - to_serializable() — enriched with variant data, prices, totals
2. Also add get_cart_for_request(request) helper that returns SessionCart or DB Cart depending on auth.
</action>
<done>[ ]</done>
</task>

<task id="03-01-03">
<name>Create cart serializers and views</name>
<action>
1. Serializers: CartItemSerializer, CartSerializer (with computed subtotal, shipping, total).
2. Views: CartView (GET cart, DELETE clear), CartItemAddView (POST add), CartItemUpdateView (PATCH quantity), CartItemDeleteView (DELETE remove), CartMergeView (POST merge session→DB).
3. URLs: wire under /api/cart/.
4. Add shipping flat rate setting: SHIPPING_FLAT_RATE = Decimal('249.00') in base.py.
</action>
