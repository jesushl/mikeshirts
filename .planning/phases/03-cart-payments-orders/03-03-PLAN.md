# Plan 03: Checkout Flow + Payments + Tickets + Email + Tracking

## Objective

Wire everything together: the checkout endpoint that converts cart into order and initiates Mercado Pago payment, the webhook that processes payment results, and the post-payment logic (stock deduction, ticket generation, email). Plus the order tracking API for customers.

## Tasks

<task id="03-03-01">
<name>Create payments app with MP integration</name>
<action>
1. Add mercadopago SDK to requirements.txt, install.
2. Payment model: order FK, mp_payment_id, mp_status, mp_status_detail, mp_payment_type, amount, raw_data (JSONField), created_at.
3. services.py — create_mp_preference(order) that builds MP preference with items, payer, back_urls, notification_url. Returns init_point.
4. Add MP settings: MERCADOPAGO_ACCESS_TOKEN (from env), MERCADOPAGO_NOTIFICATION_URL, SITE_URL.
5. Add EMAIL_BACKEND = console for dev.
</action>
<done>[ ]</done>
</task>

<task id="03-03-02">
<name>Build checkout endpoint and order creation service</name>
<action>
1. orders/services.py — create_order_from_cart(user, shipping_address_id): validates cart not empty, snapshots prices and address, creates Order + OrderItems, clears cart. Returns order.
2. orders/views.py — CheckoutView(POST): calls create_order, then create_mp_preference, returns {order_id, init_point}.
3. orders/serializers.py — CheckoutSerializer (input: shipping_address_id), OrderListSerializer, OrderDetailSerializer (with items, tickets).
</action>
<done>[ ]</done>
</task>

<task id="03-03-03">
<name>Build webhook handler with post-payment logic</name>
<action>
1. payments/views.py — WebhookView(POST): receives MP notification, fetches payment from MP API, creates/updates Payment record.
2. If payment approved: update Order status → paid, call post_payment_process(order).
3. post_payment_process: deduct stock per OrderItem (quantity available), create ProductionTicket for shortfall, generate SaleTicket (auto folio), generate ShipTicket, send confirmation email.
4. Webhook must be idempotent (check if already processed).
5. CSRF exempt for webhook endpoint.
</action>
<done>[ ]</done>
</task>

<task id="03-03-04">
<name>Build order tracking API and wire URLs</name>
<action>
1. GET /api/orders/ — list user's orders (paginated).
2. GET /api/orders/<id>/ — detail with items, sale ticket, ship ticket, production tickets.
3. Wire orders/urls.py and payments/urls.py into project urls.
</action>
<done>[ ]</done>
</task>
