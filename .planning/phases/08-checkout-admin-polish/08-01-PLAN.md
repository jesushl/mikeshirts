# Plan 08-01 — Checkout + Post-Pago

## Objective

Build the checkout flow and post-payment return pages. Install react-helmet-async for SEO in later waves.

## Context

- Checkout API: `POST /api/orders/checkout/ {shipping_address_id}` → `{order_id, init_point}`.
- `init_point` is a Mercado Pago URL — frontend redirects with `window.location.href`.
- MP back_urls configured in backend:
  - success: `/checkout/success?order=<id>`
  - failure: `/checkout/failure?order=<id>`
  - pending: `/checkout/pending?order=<id>`
- User must be authenticated and have at least one address.
- Cart is consumed server-side when order is created — frontend should clear cart after redirect.
- `GET /api/orders/<id>/` returns order detail (used on success page to show folio, items, totals).
- User addresses available in `user.addresses` from authStore.
- `useRequireAuth` hook exists to protect routes.
- Existing `CartPage` shows cart summary which can be referenced for checkout summary styling.
- Analytics: `POST /api/analytics/event/ {event_type: 'checkout_start'}`.
