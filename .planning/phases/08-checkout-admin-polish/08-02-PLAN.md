# Plan 08-02 — Mis Pedidos

## Objective

Build the customer-facing order pages: list of past orders and order detail with tickets and status tracking.

## Context

- `GET /api/orders/` → `{count, results: [{id, status, status_display, total, item_count, created_at}]}`.
- `GET /api/orders/<id>/` → full detail with items[], sale_ticket (folio, issued_at), ship_ticket (tracking, carrier, status), production_tickets[], shipping_address (JSON snapshot), totals.
- Order statuses: pending_payment, paid, in_production, shipped, delivered, cancelled.
- ShipTicket statuses: pending, shipped, delivered.
- ProductionTicket statuses: pending, in_production, completed.
- Types already defined in 08-01 (OrderListItem, OrderDetail, etc.).
- Orders service (fetchOrders, fetchOrder) already created in 08-01.
- `formatMXN` and `useRequireAuth` available.
