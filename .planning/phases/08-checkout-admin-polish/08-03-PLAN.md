# Plan 08-03 — Panel Admin

## Objective

Build a full-featured, polished admin panel at `/gestion/*` with role-based dashboards for ventas, producción, envíos, and analítica.

## Context

- Admin API endpoints all under `/api/admin/` with role-based permissions (ventas, envios, produccion, admin_full).
- User groups stored in backend; frontend needs to check user.groups or rely on 403 responses.
- Current User type doesn't include groups — need to either add it or determine role from API.
- Design should match store quality (full_custom decision from discuss-phase).
- Paleta: #0a0a0a fondo, #e11d48 accent, #06b6d4 cyan, #f5f5f5 texto, #141414 surface.
- Dashboard data from `GET /api/admin/dashboard/`: orders_today, revenue_today, pending_orders, pending_production, in_production, pending_shipping, low_stock_count.
- Ventas metrics from `GET /api/admin/ventas/metrics/?days=30`: total_sales, order_count, avg_ticket.
- Analytics from `GET /api/admin/analytics/?days=30`: visits, pageviews, funnel, conversion_rate, top_referrers, top_utm.

   - `fetchAnalytics(days?)` → analytics dashboard data.
